# 🏥 Sistema de Gestión Hospitalaria (POO Java)

Este proyecto es una simulación de un entorno hospitalario desarrollado para demostrar el dominio de la **Programación Orientada a Objetos (POO)** en Java. Se han aplicado conceptos avanzados de arquitectura de software para crear un código limpio, escalable y profesional.

## 📊 Puntos Clave del Proyecto

- **Abstracción e Interfaces (20%):** Uso de `IDoctor` e `IPatient` para definir contratos de comportamiento y una clase abstracta `Characters` como base.
- **Encapsulamiento (15%):** Atributos privados con métodos Getters y Setters para un acceso seguro a la información.
- **Herencia (15%):** Estructura jerárquica que permite a las subclases especializarse a partir de una superclase común.
- **Principios SOLID (10%):** Aplicación de Responsabilidad Única y Segregación de Interfaces.
- **Polimorfismo:** Capacidad de gestionar diferentes tipos de personal y pacientes bajo una misma interfaz.

## 🛠️ Requisitos de Instalación
1. Clonar el repositorio.
2. Abrir el proyecto en **IntelliJ IDEA**.
3. Asegurarse de tener configurado el **JDK 11** o superior.
4. Ejecutar la clase `Main.java` para ver la simulación en consola.
